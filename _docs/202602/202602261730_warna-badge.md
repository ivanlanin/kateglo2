# Dokumentasi Warna Badge

## Tujuan

Dokumen ini menjadi acuan warna badge di Kateglo 2.0 agar konsisten antar halaman publik dan redaksi, termasuk mode terang dan gelap.

## Sumber Utama

- Gaya global badge: `frontend/src/styles/index.css`
- Badge status admin umum: `frontend/src/komponen/redaksi/KomponenAdmin.jsx`
- Badge status audit makna: `frontend/src/halaman/redaksi/AuditMaknaAdmin.jsx`

## Prinsip Umum

- Gunakan utilitas Tailwind (`bg-*`, `text-*`) yang sudah dipakai saat ini.
- Untuk badge kategori/makna, gunakan pola kelas dasar `kamus-badge` + kelas variasi warna.
- Untuk mode gelap, gunakan pasangan kelas `.dark` yang setara semantik (bukan warna baru ad hoc).
- Hindari inline style untuk badge.

## Aturan Penambahan Badge Baru

Jika menambah badge baru:

1. Definisikan style di `frontend/src/styles/index.css` (untuk badge publik/umum).
2. Tetapkan pasangan warna light/dark yang semantik dan kontras.
3. Gunakan kelas dasar yang sudah ada bila relevan (mis. `kamus-badge`).
4. Tambahkan entri baru ke dokumen ini agar peta warna tetap sinkron.
